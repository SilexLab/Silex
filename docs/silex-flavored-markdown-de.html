Silex Flavored Markdown (SFM) ist eine auf Markdown basierende erweiterte Markupsprache, entworfen für das Silex Framework und die darauf basierende Bulletin Board Software Silex Bulletin Board.

Diese Markupsprache besitzt folgende Syntax:

  • Überschriften

    Überschriften sind in den 2 Stilen von Markdown und einer veränderten Form von atx möglich, Setext und atx.

    Überschriften im Setextstil werden unterstrichen:

    Überschrift Ebene 1 <h1>
========================

Überschrift Ebene 2 <h2>
------------------------

Überschrift Ebene 3 <h3>
************************

    Die Anzahl der Unterstriche ('=', '-' und '*') ist unbedeutend, jede Anzahl wird funktionieren.

    Überschriften im atx-stil verwenden 1-6 Rautezeichen am Anfang der Zeile, die Anzahl der Rauten steht dabei für die Ebene der Überschrift:

    # Überschrift Ebene 1 <h1>

## Überschrift Ebene 2 <h2>

### Überschrift Ebene 3 <h3>

###### Überschrift Ebene 6 <h6>

    Optional können noch Rauten an das Ende der Zeile gehängt werden, die Anzahl dieser ist egal.

    Eine weitere Möglichkeit ist diese abgewandelte Variante des atx-stils:

    #1 Überschrift Ebene 1 <h1>

#2 Überschrift Ebene 2 <h2>

#3 Überschrift Ebene 3 <h3>

#6 Überschrift Ebene 6 <h6>

    Bei dieser Methode wird am Anfang der Zeile eine Raute geschrieben, gefolgt von der Zahl (1-6), welche die Überschriftebene wiederspiegelt (wobei Ebene 1 - 2 keinen verkürzenden Nutzen im vergleich zu atx bringen).

  • Blockzitate

    Silex Flavored Markdown benutzt den in Markdown benutzten "email-style" für Zitate:

    > Lorem ipsum dolor sit amet, consetetur sadipscing elitr,
> sed diam nonumy eirmod tempor invidunt ut labore et dolore magna aliquyam erat,
> sed diam voluptua.
>
> At vero eos et accusam et justo duo dolores et ea rebum.

    Es ist ebenfalls möglich nur ein '>' am Anfang der Zeile vor einem Absatz zu schreiben:

    > Lorem ipsum dolor sit amet, consetetur sadipscing elitr,
sed diam nonumy eirmod tempor invidunt ut labore et dolore magna aliquyam erat,
sed diam voluptua.

> At vero eos et accusam et justo duo dolores et ea rebum.

    Bei dieser Methode entstehen 2 Zitate. Um beide Zitate zu verbinden, muss in die Leerzeile ein '>' gesetzt werden.
    Es ist auch möglich Zitate zu verschachteln: 

    > Erste Zitatebene
>
> > Zweite Zitatebene
>
> Erste Zitatebene

    Blockzitate können ebenfalls alle anderen Markupelemente dieses codes enthalten.

    Mit Silex Flavored Markdown ist es möglich den Zitatursprung festzulegen:

    [Name der Zitierten Person, http://Link-zum-Zitierten.com/]
> Lorem ipsum dolor sit amet, consetetur sadipscing elitr,
    > sed diam nonumy eirmod tempor invidunt ut labore et dolore > magna aliquyam erat,
> sed diam voluptua.

    Hierbei wird über dem Zitatblock eine in eckigen Klammern geschachtelte Zeile geschrieben, in der die Informationen über den Namen der Person, von der das Zitat stammt und optional einen Link dazu stehen.

  • Listen

    Unterstützt werden wie in Markdown geordnete und ungeordnete Listen. Als Auflistungszeichen für unsortierte Listen können '*', '+' und '-' verwendet werden. Verschiedene Zeichen können jedoch nicht in einer einzigen Liste gemischt verwendet werden, es würde immer eine neue Liste entstehen.

    * Rot
* Grün
* Blau

    ist das gleiche wie:

    - Rot
- Grün
- Blau

    und:

    + Rot
+ Grün
+ Blau

    Aus folgendem würden demnach 3 Listen werden:

    * Rot
- Grün
+ Blau

    Verschachteln von Listen ist durch das Aufrücken der Listen auf den Text der vorherigen Ebene möglich:

    * Erste Listenebene
	* Zweite Listenebene
* Wieder die erste Listenebene
	* Zweite Listenebene
	* Auch die zweite
		* Dritte Listenebene
* Erste Listenebene

    Eine Liste wird durch eine Leerzeile verlassen

    Eine sortierte Liste kann folgendermaßen Angelegt werden:

    1. Rot
2. Grün
3. Blau

    Um Buchstaben statt Zahlen als Auflistungszeichen zu verwenden, müssen lediglich Buchstaben verwendet werden:

    A. Rot
B. Grün
C. Blau

    Das Verschachteln ist bei sortierten Listen ebenfalls möglich. Zahlen und Buchstaben Auflistungen können nicht gemischt werden.

    Statt die Zeilen durch zu nummerieren, kann auch einfach eine beliebige Zahl oder ein beliebiger Buchstabe verwendet werden:

    1. Rot
1. Grün
1. Blau

3. Rot
1. Grün
9. Blau

A. Rot
A. Grün
A. Blau

    Um zu erzwingen, dass sortierte Listen mit der/dem gewählten Auflistungszahl/-buchstaben aufgelistet werden, muss an das Ende der/des Zahl/Buchstabes ein Backslash '\' angehängt werden.

    In Listen können - wie in Blockzitaten - alle "Silex flavored Markdown" Markup Elemente verwendet werden.

  • Code(blöcke)

    Codeblöcke werden mit zwei eckigen Klammern geöffnet vor und geschlossen nach dem eigentlichen Code geschrieben:

    [[
<?php
public function hello() {
	echo 'Hello world!';
}
?>
]]

    4 Zeichen im Codeblock sind ein Tab. Um ein Syntaxhighlighting durchzuführen, muss vor die zwei eckigen Klammern die hervorzuhebende Sprache in eckige Klammern geschrieben werden:

    [PHP][[
<?php
// Comment
public function hello() {
	echo 'Hello world!'.CONST;
}
?>
]]

    wird dann zu:

    <?php
// Comment
public function hello() {
	echo 'Hello world!'.CONST;
}
?>

    Über dem Codeblock kann dann die gewählte Syntax stehen.

    Vorformatierte Texte in einer Zeile (zwischen normalem Text) (inline) wird folgendermaßen erzeugt:

    [[ echo 'Hello world!'; ]]

    Dies erzeugt einen <code>-Tag, dieser kann wie folgt aussehen:

    echo 'Hello world!';

    Syntaxhighlighting ist ebenfalls möglich, indem die entpsprechende Sprache vor dem Block angegeben wird:

    [PHP][[ echo 'Hello world!'; ]]

    Dies erzeugt einen <code>-Tag mit hervorgehobenem Code:

    echo 'Hello world!';

    Code in inline- und Codeblöcken der Silex Flavored Markdown enthält, wird nicht geparst.

  • Horizontale Linie

    Eine Horizontale Linie (<hr>) kann mit folgenden Zeilen erzeugt werden:

    * * *

***

*****

- - -

--------------------

    Die Anzahl der Leerzeichen und/oder Zeichen kann beliebig gewählt sein, die Zeichanzahl ('*' oder '-') muss nur größer oder gleich 3 sein.

    Top

  • Hyperlinks

    Hyperlinks werden wie in Markdown mit einem eckigen Klammernpaar, in dem der klickbare Text des Hyperlinks steht gefolgt von einem runden Klammernpaar, in dem der Hyperlink steht erzeugt:

    [Dies ist der Hyperlinktext](http://example.com/ "Linkbeschreibung")

    Die Linkbeschreibung und der Hyperlinktext sind beide optional, ist der Hyperlinktext nicht gesetzt, wird als Text das Ziel des Links genommen:

    [](http://example.com/)
[Text](http://example.com/)
    Top

  • Bilder

    Auch das einbinden von Bildern funktioniert ähnlich wie in Markdown:

    ![Alternativer Text](http://exampe.com/image.png "Titel" 256x128px)

    Der Alternative Text, Titel - welcher beim hovern angezeigt wird - und die Größe sind optional.
    Die Größe ist neu hinzu gekommen und bietet die Möglichkeit die Höhe, Breite oder Beides des eingebetteten Bildes zu verändern (Damit wird lediglich die Anzeigegröße verändert):

    ![](http://example.com/image.png w200px)

    Dieses Beispiel legt lediglich die Breite (w) des Bildes fest, die Höhe (h) wird automatisch im passenden Seitenverhältnis mit geändert.

  • Textformatierungen

    Standardformatierungen

    Diese Markupsprache bietet verschiedene Möglichkeiten den Text hervorzuheben. Das '*'-Zeichen und das '_'-Zeichen kommen dabei zum Einsatz:

    *Einfaches Sternchen*
**Doppeltes Sternchen**
***Dreifaches Sternchen***
_Unterstrichen_
_*Einfaches Sternchen unterstrichen*_

    aus diesem Beispiel wird folgendes:

    Einfaches Sternchen
    Doppeltes Sternchen
    Dreifaches Sternchen
    Unterstrichen
    Einfaches Sternchen unterstrichen

    Schachtelungen sind ebenfalls, aber nur auf das höhere Element (Anzahl Sterne) möglich:

    *Kursiv **Fett** ***Fett und Kursiv*** und nur noch _Kursiv mit Unterstreichung_*
** Fett ***Fett und Kursiv*** und *Kursiv* nur _Fett_**

    Aus diesem Beispiel wird:

    Kursiv Fett Fett und Kursiv und nur noch Kursiv mit Unterstreichung
    Fett Fett und Kursiv und Kursiv nur Fett

    Erweiterte Formatierungen

    Als weitere Formatierungen gibt es folgende:

    -(Durchgestrichen)-
Ref^(Hochgestellt)^
Ref,(Tiefgestellt),

    Diese werden so geparst:

    Durchgestrichen
    RefHochgestellt
    RefTiefgestellt

    Textfarbe

    Die Textfarbe kann wie folgt geändert werden:

    Normaler Text und [color:#3A8000][Text in Bunt und in Farbe] bis hier. [c:red][Shortcut]

    Wird zu:

    Normaler Text und Text in Bunt und in Farbe bis hier. Shortcut

    Schachtelungen von Farben sind nicht möglich. Statt eines 3 oder 6 stelligen hexadezimalen Farbcodes sind auch noch Farbbezeichnungen wie red oder white möglich.

    Textgröße

    Die Textgröße wird per [size=##px][Größe] geändert:

    Normaler und [size:20px][großer Text]. [s:18px][Shortcut]

    Wird zu:

    Normaler und großer Text. Shortcut

    Textgrößen können ebenfalls nicht geschachtelt werden.

    Schriftart

    Die Schriftart kann wie folgt festgelegt werden:

    Normale und [font:Comic Sans][eine hässliche Schriftart]. [f:Comic Sans][Shortcut]

    Wird zu:

    Normale und eine hässliche Schriftart. Shortcut

    Zusammenfassen

    Das aneinanderreihen, der Attribute ist ebenfalls möglich:

    [color:blue,s:18px,f:Comic Sans][Aneinandergereiht]

    wird zu geparst:

    Aneinandergereiht

  • Textformatierungen in Wörtern (trivial)

    Unterstriche in Wörtern unterstreichen das Wort nur, so bald dieses mit eckigen Klammern festgelegt wurde. Aus

    Lorem_ipsum_dolor_sit_amet.
Lorem*ipsum*dolor***sit***amet.

    wird dann:

    Lorem_ipsum_dolor_sit_amet.
    Lorem*ipsum*dolor***sit***amet.

    und

    Lorem[_]ipsum[_]dolor[_]sit[_]amet.
Lorem[*]ipsum[*]dolor[***]sit[***]amet.

    wird zu:

    Loremipsumdolorsitamet.
    Loremipsumdolorsitamet.

    Alle anderen Formatierungen funktionieren auch in Wörtern.

    Top

  • Spoiler

    Spoiler können wie folgt erzeugt werden:

    [spoiler][Verborgener Plot]

  • Backslash escape

    Um Zeichen, die in dieser Markupsprache eine Funktion haben trotzdem schreiben zu können, müssen diese per Backslash escaped werden:

    Lorem *ipsum \*dolor\* sit* amet.

    Dies weist dem Parser an, diese Textstellen zu ignorieren. Aus dem obigen Beispiel wird folgender Text:

    Lorem ipsum *dolor* sit amet.
